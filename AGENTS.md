# AGENTS.md

Vynody — cross-platform local music player (Flutter + Audio Core/ExoPlayer audio kernels).

## Build & dev commands

```bash
flutter pub get                          # install dependencies
dart run build_runner build --delete-conflicting-outputs  # codegen (freezed, json_serializable, drift)
flutter analyze                          # static analysis + lint
flutter test                             # unit & widget tests (test/)
flutter test integration_test/           # integration tests
flutter run -d <device-id>               # dev run on a device
```

## Pre-check order on any change

1. `dart run build_runner build --delete-conflicting-outputs` — if you touch model/DAO files that use `@freezed`, `@JsonSerializable`, or Drift tables.
2. `flutter analyze` — must pass (generated `*.g.dart` / `*.freezed.dart` are excluded from analysis).
3. `flutter test` — run before pushing.

## Architecture notes

- **State management:** Riverpod (`flutter_riverpod` v3). Providers live co-located with their domain code (e.g., `lib/player/audio/` has audio providers).
- **Database:** Drift (SQLite). Schema defined via Dart annotations in `lib/player/metadata/`. After changing a table, run `build_runner`.
- **Audio kernels are platform-specific.** Do NOT assume one audio API works everywhere:
  - Windows/Linux/macOS/iOS: Rust/Native audio kernel (`audio_core` submodule; Apple platforms integrate AVFoundation + FFmpeg fallback)
  - Android: ExoPlayer (`lib/player/platform/` & `audio_core`)
- **Key directories:**
  - `lib/player/` — core player logic (audio engine, lyrics, metadata, scanner, settings, sharing)
  - `lib/pages/` — UI pages/tabs
  - `lib/widgets/` — reusable widgets
  - `lib/models/` — data models (mostly freezed)
  - `lib/l10n/` — ARB localization files (zh + en), auto-generated via `generate: true`
- **Codegen is essential.** Many types are generated by `freezed` (immutable classes), `json_serializable` (JSON), and `drift_dev` (DB). On a fresh clone or after dependency changes, run `build_runner` before `analyze`.

## Gotchas

- **`audio_core` is a git submodule.** Clone with `--recurse-submodules` or run `git submodule update --init`. It's used via `dependency_overrides: audio_core: path: ./audio_core` — always from the local checkout, not pub.dev.
- **Multiple `dependency_overrides`** in `pubspec.yaml` override upstream packages (`bonsoir`, `bonsoir_windows`, `permission_handler_windows`, `audio_core`). These exist for fork compatibility; don't remove them without understanding why.
- **Rust toolchain required** for Windows/Linux desktop builds and Android cross-compilation. The `audio_core` plugin contains native Rust code built via Cargokit.
- **`flutter analyze` excludes generated files** (`analysis_options.yaml`), so codegen errors won't appear until runtime. Always run `build_runner` after touching annotated classes.
- **No pre-commit hooks or task runner** — run `flutter analyze` and `flutter test` manually before pushing.
- **Android keystore** is a CI secret (`ANDROID_KEYSTORE_BASE64`); local debug builds use the default debug keystore.
